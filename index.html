Investigative Approach

The projects are organized around operational reasoning: what happened, what evidence supports it, what visibility existed, what gaps were found, and what changed as a result.

Operational Troubleshooting Built and maintained the lab infrastructure behind the portfolio, including Debian, KVM/QEMU, pfSense, Active Directory, Windows endpoints, and Splunk. The write-ups show configuration choices, failures, root causes, and fixes so the troubleshooting process is visible alongside the final result.
Telemetry Validation Configured Windows Event Logs, Sysmon, Splunk Universal Forwarder, and Windows Filtering Platform auditing, then verified what data actually appeared in Splunk before relying on it for detection.
Evidence Based Investigation Used SIEM data and vulnerability findings to reconstruct activity, explain impact, document likely attacker goals, and identify what a defender would need to monitor or tune.
Process Discipline Documented gaps, false positives, alert delays, authentication issues, firewall drift, and remediation steps so each project shows not just what worked, but what had to be corrected along the way.
