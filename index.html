What this portfolio proves

This is not a tool screenshot museum. The work is organized around investigation: what happened, what evidence supports it, what an attacker could do next, what a defender would see, and what needed to be tuned or remediated.

Built the environment Debian 13 host, KVM/QEMU virtualization, pfSense segmentation, Windows Server 2022 domain controller, Windows 11 workstation, Ubuntu Splunk server, and Kali attack VM.
Instrumented telemetry Splunk Universal Forwarder, Windows Security logs, System/Application logs, Sysmon operational logs, and Windows Filtering Platform audit events.
Validated visibility Found that network scans were invisible until WFP auditing was enabled, then confirmed scan signatures in Splunk through repeat testing.
Documented failures Timezone mismatches, scheduler backlog, queue behavior, GVM authentication issues, Windows firewall profile drift, and Linux firewall rule conflicts are documented instead of quietly buried like inconvenient fossils.
